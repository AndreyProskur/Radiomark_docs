Технологии хранения данных

Каждая сущность платформы Radiomark предъявляет разные требования к объёму, консистентности, паттернам доступа и стоимости хранения. На основе этих требований выбраны два основных типа хранилищ: реляционная СУБД (PostgreSQL) и объектное хранилище (S3/MinIO). Для быстрого чтения используется Redis, а для гибких данных — JSONB внутри PostgreSQL.

Реляционная СУБД (PostgreSQL)​

👤 Пользователи

Объём: ~10 000 записей (< 1 ГБ)

Строгая структура, частые чтения при аутентификации, высокие требования к целостности (ACID).

📋 Заявки пациентов

Объём: десятки тысяч записей (~10 ГБ)

Частые обновления статусов, связи с пользователями и снимками, обязательные транзакции.

🔬Медицинские снимки (метаданные)

Объём: десятки тысяч записей, десятки ТБ с файлами

Критичные данные со сложными связями. Требуется ACID и поддержка сложных запросов.

📝 Элементы разметки

Объём: 30–100 тысяч записей

Жёсткая связь со снимком. Для координат используется гибкий JSONB.

📄 Медицинские документы (метаданные)

Объём: соответствует количеству заявок

Строгая структура, связь с заявкой и врачом. Файлы документов вынесены в S3.

⭐ Отзывы

Объём: десятки тысяч записей (< 1 ГБ)

Фиксированный набор полей, фильтрация по врачу и пациенту, минимальная стоимость хранения.

📊 Прогресс студента

Объём: одна запись на студента

Аналитические данные с частым чтением. Для ускорения используется Redis.

Объектное хранилище (S3/MinIO)​

⏳ Временные снимки

Объём: ~1000 файлов единовременно (до 100 ГБ)

Файлы хранятся временно (TTL 24 часа). Метаданные — в PostgreSQL. Дешёвое и масштабируемое решение.

📁 Файлы снимков и документов

Объём: десятки ТБ

Большие бинарные файлы хранятся в S3/MinIO, в БД — только ссылки. Обеспечивает масштабируемость и снижает стоимость.

Почему именно эти технологии?​

🐘 PostgreSQL

Гарантирует транзакции (ACID), поддерживает сложные SQL-запросы и индексы. Отлично подходит для структурированных данных с высокими требованиями к целостности.

🗂️ JSONB в PostgreSQL

Гибкое хранение координат разметки и клинических параметров снимка без частых миграций схемы. При этом сохраняется возможность индексации и поиска.

☁️ Объектное хранилище (S3/MinIO)

Дешёвое и масштабируемое хранение больших бинарных файлов. Поддерживает политики жизненного цикла (TTL) и CDN для быстрой доставки изображений.
