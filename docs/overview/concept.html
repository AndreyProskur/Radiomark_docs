Концепция и границы продукта

Проблемы, которые решает Radiomark​

Почему Radiomark?
  • Фрагментированные знания – студентам и начинающим врачам не хватает единой базы качественно размеченных снимков.
  • Долгое ожидание диагностики – пациенты вынуждены ждать описания снимков в государственных клиниках или переплачивать в частных.
  • Дефицит медицинских датасетов – ML‑инженеры тратят время на поиск и верификацию данных, часто получая сомнительное качество разметки.

Целевая аудитория​

СегментПотребности
Студенты / начинающие рентгенологиРазвитие насмотренности, тренировка на реальных кейсах
ПациентыБыстрое и доступное заключение по снимку
ML‑инженерыГотовые к обучению моделей датасеты без дополнительной обработки
Врачи‑рентгенологиЭкспертная работа, разметка, модерация, дополнительный доход

Ключевые метрики (KPI)​

Цели проекта
  • 5 000 активных пользователей (студенты + врачи) через 12 месяцев после запуска
  • 20 000 размеченных снимков в базе через 18 месяцев
  • 5 B2B‑сделок с клиническими центрами на продажу ML‑датасетов через 24 месяца
  • Сокращение времени обработки снимка на 20% за первые 6 месяцев работы

План развития (Roadmap)​

Roadmap
  • MVP (0–6 мес.) – веб‑платформа, регистрация, загрузка, разметка экспертами, тренировочная разметка, поиск по категориям
  • Версия 2.0 (6–12 мес.) – API для интеграции, поиск по клиническим диагнозам, система рейтинга врачей, улучшенная разметка
  • Версия 3.0 (12–24 мес.) – интеграция с медицинскими информационными системами, международная локализация, мобильное приложение

Границы текущей версии (MVP)​

Включено в MVP:

  • Загрузка и хранение снимков
  • Анонимизация данных пациентов
  • Поиск и фильтрация снимков по категориям
  • Личные кабинеты (врач, студент, пациент, ML‑инженер)
  • Базовая система экспорта датасетов
  • Интерактивная тренировочная разметка
  • Инструменты разметки для врачей с возможностью прикреплять заключения

НЕ включено в MVP:

  • Полноценный API (появится в v2.0)
  • Мобильное приложение (v3.0)
  • Интеграция с ЕМИАС и другими МИС (v3.0)
  • Международная локализация (v3.0)
  • Инструменты разметки с ИИ

Ключевые риски​

  • Юридические риски (152‑ФЗ, персональные данные) – полная анонимизация, согласия, защищённые серверы
  • Нехватка экспертов – партнёрства с клиниками и вузами, доля от дохода
  • Конкуренция – упор на удобство, экспертность, скорость обработки
  • Технические риски – переход на микросервисную архитектуру, горизонтальное масштабирование

Сроки​

  • 0–2 мес. – проектирование архитектуры, UX/UI-дизайн, подготовка юридических документов
  • 2–5 мес. – разработка основы системы, загрузка и хранение снимков, личные кабинеты
  • 5–6 мес. – тестирование, пилотный запуск, подключение врачей

Ресурсы​

  • Люди: врачи, PM/PO, DevOps, UX/UI-дизайнер, юристы, backend- и frontend-разработчики, системный аналитик
  • Технологии: база данных, хранилище изображений, инструменты разметки, веб-фреймворк, инструменты анонимизации
  • Инструменты: Git, Jira, Confluence, UML, BPMN, VS Code, Docker, СУБД
