# Концепция и границы курсового проекта

## Рентгенологическая платформа Radiomark

### Бизнес-требования

#### 1. Описание проблемы

**Проблемы:**
Мой проект создан, чтобы решить сразу несколько проблем:
- Создание большой единой базы данных с множеством рентгенологических снимков по разным категориям, по которой молодые специалисты смогут улучшать свою практическую насмотренность и тренироваться в описаниях снимков.
- Ускорение процесса описания и анализа снимков для пациентов — пациенты смогут получить быстрое заключение по их снимку от врачей-рентгенологов.
- Создание датасетов из снимков для тренировки различных ИИ-инструментов. Специалистам не нужно будет искать на разных платформах датасеты с сомнительным качеством разметки.

**Текущие альтернативы:**
- Начинающие врачи используют учебники или снимки из лечебных заведений, в которых работают.
- Пациенты ждут описания снимков в государственных больницах или уходят в частные клиники, где могут переплачивать.
- ML-специалисты ищут датасеты в открытом доступе и часто отдают их на повторную разметку рентгенологам.
- Существуют похожие проекты, например, **Rentgenogram**.

Но все эти решения недостаточны, потому что на ожидание или поиск информации тратится много времени, а в существующих базах данных мало снимков, которые к тому же часто не содержат разметки и описания специалиста.

#### 2. Целевая аудитория

**Сегменты аудитории:**
- Студенты / начинающие рентгенологи.
- Пациенты с необработанными рентгенологическими снимками.
- ML-инженеры, которым нужны качественные датасеты.
- Опытные врачи-рентгенологи для экспертной разметки снимков и составления заключений пациентам.

**Потребности и ожидания:**
- **Студенты** ожидают полноценную базу знаний для развития насмотренности и лучшего понимания рентгенологических снимков.
- **Пациенты** ожидают значительного сокращения времени обработки их снимков.
- **ML-инженеры** ожидают базу датасетов, готовых к обучению моделей без дополнительных действий.

#### 3. Стейкхолдеры

**Ключевые стейкхолдеры:**
- Владельцы и инвесторы проекта.
- Команда разработки.
- Врачи-рентгенологи.
- Клиенты (студенты, пациенты, ML-инженеры).
- Медицинские учреждения.
- Регуляторные медицинские органы.

**Роль стейкхолдеров:**
- **Владельцы и инвесторы** — определяют стратегию, финансируют, контролируют ключевые метрики.
- **Команда разработки** — отвечает за техническую реализацию, архитектуру и стабильность платформы.
- **Врачи-рентгенологи** — проводят описание и разметку снимков, составляют заключения.
- **Клиенты** — используют платформу, дают обратную связь, приносят доход.
- **Медицинские учреждения** — предоставляют снимки, выступают потенциальными клиентами для обучения персонала.
- **Регуляторные органы** — контролируют соблюдение медицинских нормативов и сохранность персональных данных.

#### 4. Бизнес-цели и ключевые метрики

**Основные бизнес-цели:**
- За 12 месяцев привлечь и удержать минимум 5 000 активных пользователей (студентов и врачей) с соотношением опытные врачи / студенты = 30% / 70%.
- За 18 месяцев создать базу из 20 000 размеченных рентгенологических снимков.
- За 24 месяца заключить минимум 5 B2B-сделок с клиническими центрами на продажу ML-датасетов с суммарной выручкой более 7 000 000 рублей.
- За 6 месяцев сократить время обработки снимков на 20% по сравнению с первым месяцем работы.

**Ключевые метрики (KPI):**
- Количество зарегистрированных пользователей.
- Удержание пользователей (Retention rate).
- Количество размеченных снимков.
- Среднее время обработки одного снимка.
- Количество жалоб пациентов (отслеживание удовлетворённости).
- Выручка компании.

### Описание продукта

**Общая концепция:**
Radiomark — цифровая рентгенологическая платформа, объединяющая:
- Образовательную базу размеченных снимков для студентов.
- Сервис ускоренной обработки снимков и получения заключения для пациентов.
- Источник качественных датасетов для обучения ИИ.

Основная ценность для разных групп: развитие насмотренности и навыков у студентов, быстрая и доступная экспертная оценка для пациентов, готовые к обучению ИИ данные с разметкой для ML-инженеров.

**Ключевые функции:**
- Сохранение рентгенологических снимков с разметкой по категориям.
- Экспертная разметка снимков и составление заключения.
- Экспорт датасетов для ML.
- Ускоренное получение заключения по снимку.
- Удобный поиск снимков по категориям, заболеваниям и т.д.

### Риски

- **Юридические риски** (персональные данные, права пациентов) → полная анонимизация снимков, добавление только по согласию, защищённые серверы.
- **Нехватка экспертов** → партнёрские программы с клиниками и институтами, мотивация долей от дохода.
- **Конкуренция** → упор на удобство, экспертность, УТП в виде ускоренной обработки снимков.
- **Технические сложности** (хранение, масштабирование) → переход на микросервисную модель, горизонтальное масштабирование серверов.

### Границы и ограничения

#### Основные функции продукта (MVP)

**В первичной версии будут реализованы:**
- Загрузка и хранение рентгенологических снимков.
- Анонимизация данных пациентов.
- Поиск и фильтрация снимков по категориям / заболеваниям.
- Личные кабинеты пользователей (врач, студент, пациент, ML-инженер).
- Базовая система экспорта датасетов.
- Интерактивное окно для тренировочной разметки и описания снимка (для студентов).
- Интеграция инструментов для разметки снимков врачами-рентгенологами и возможность прикреплять заключения в виде документов.

**Функции, которые НЕ будут реализованы в текущей версии:**
- Полноценный API для интеграции.
- Мобильное приложение.
- Интеграция с медицинскими информационными системами (например, ЕМИАС).
- Международная локализация.
- Улучшенные инструменты разметки (возможно, с применением ИИ).

#### Версии продукта и план развития

- **MVP (0–6 месяцев):** веб-платформа, регистрация, разметка экспертами, тренировочная разметка, загрузка снимков, сортировка и поиск по категориям.
- **Версия 2.0 (6–12 месяцев):** API для интеграции, поиск по клиническим диагнозам, улучшенная разметка, система рейтинга врачей.
- **Версия 3.0 (12–24 месяца):** интеграция с МИС, международная локализация, мобильное приложение.
