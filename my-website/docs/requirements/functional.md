# Функциональные требования

## UC-01: Регистрация пользователя

| Поле | Описание |
|------|----------|
| **ID** | UC-01 |
| **Название** | Регистрация пользователя |
| **Акторы** | Незарегистрированный пользователь сервиса, Система |
| **Предусловия** | Пользователь не зарегистрирован на платформе<br />У пользователя есть интернет-соединение<br />Пользователь находится на странице регистрации |
| **Постусловия (Успех)** | Учетная запись пользователя создана |
| **Основной поток** | 1. Пользователь заполняет регистрационную форму (логин, пароль, email и роль)<br />2. Пользователь подтверждает согласие с политикой конфиденциальности<br />3. Пользователь инициирует процесс регистрации<br />4. Система проверяет корректность введенных данных (формат email, уникальность email, надежность пароля)<br />5. Система сохраняет нового пользователя в базе данных<br />6. Система отображает пользователю уведомление об успешной регистрации |
| **Альтернативный поток** | **Регистрация при выборе роли рентгенолог**<br />Точка входа: Во время шага 1<br />1.1 Система показывает дополнительное поле для загрузки документа, подтверждающего квалификацию врача<br />1.2 Пользователь загружает документ<br />1.3 Система проверяет размер и формат документа<br />Use case продолжается с шага 2<br /><br />**Просмотр политики конфиденциальности**<br />Точка входа: Во время шага 2<br />2.1 Пользователь переходит по ссылке политики конфиденциальности<br />2.2 Система отображает окно с политикой конфиденциальности пользователю<br />2.3 Пользователь закрывает окно<br />Use case продолжается с шага 3 |
| **Исключительный поток** | **Ошибка сети при отправке формы**<br />Точка входа: Во время шага 3<br />3.1 Система не получает данные формы из-за сбоя системы<br />3.2 Система выводит уведомление пользователю об ошибке загрузки данных<br />3.3 Система предлагает пользователю повторить загрузить данные заново<br />3.4 Пользователь может повторить попытку загрузить данные<br />3.5 Use case завершается<br /><br />**Email уже зарегистрирован в системе**<br />Точка входа: Во время шага 4<br />4.1 Система обнаруживает, что указанный email уже зарегистрирован в системе<br />4.2 Система отображает уведомление о наличие зарегистрированного пользователя с таким email и предлагает восстановить пароль<br />4.3 Сценарий завершается неуспешно: регистрация не выполнена (пользователь может либо закрыть форму регистрации, либо инициировать восстановление пароля) |
| **Бизнес правила** | BR-01: Пароль должен содержать минимум 8 символов<br />BR-02: Email должен быть уникальным<br />BR-03: Роль фиксируется при регистрации<br />BR-04: Для подтверждения врачебной квалификации пользователь может загрузить документ формата PDF, PNG; размером не более 5 Мб |

## UC-02: Просмотр рентгеновского снимка из учебной базы данных

| Поле | Описание |
|------|----------|
| **ID** | UC-02 |
| **Название** | Просмотр рентгеновского снимка из учебной базы данных |
| **Акторы** | Студент, Система |
| **Предусловия** | Пользователь авторизован<br />У пользователя есть интернет-соединение<br />В базе имеются опубликованные и обработанные рентгенологические снимки<br />Пользователь находится на главной странице сервиса |
| **Постусловия (Успех)** | Снимок отображён пользователю вместе с разметкой и описанием<br />Система фиксирует факт открытия снимка в истории действий пользователя<br />Если пользователь находился на странице снимка не менее установленного времени, снимку присваивается статус «Просмотрен» |
| **Основной поток** | 1. Пользователь открывает раздел "База снимков"<br />2. Система отображает список доступных категорий снимков<br />3. Пользователь выбирает категорию снимков<br />4. Система отображает список снимков выбранной категории<br />5. Пользователь выбирает конкретный снимок<br />6. Система проверяет права доступа пользователя к выбранному снимку<br />7. Система отображает выбранный снимок вместе с его разметкой и текстовым описанием<br />8. Пользователь просматривает снимок с его разметкой и описанием<br />9. Система фиксирует факт открытия снимка в истории действий пользователя<br />10. Если пользователь находится на странице не менее установленного времени, система присваивает снимку статус "Просмотрен" |
| **Альтернативный поток** | **Нет доступа к снимку**<br />Точка входа: Во время шага 6<br />6.1 Система определяет, что выбранный снимок доступен только пользователям с премиум-подпиской<br />6.2 Система отображает уведомление о недоступности этого снимка для данного пользователя<br />6.3 Система предлагает пользователю следующие действия:<br />- сменить уровень доступа к снимкам (оформить премиум-подписку)<br />- вернуться к списку снимков<br />6.4 Пользователь выбирает вернуться к списку снимков<br />6.5 Система отображает список снимков выбранной категории<br />Use case продолжается с шага 5 |
| **Исключительный поток** | **Ошибка отображения снимка**<br />Точка входа: Во время шага 7<br />7.1 Система не может получить данные снимка из хранилища<br />7.2 Система выводит уведомление об ошибке загрузки данных<br />7.3 Система предлагает пользователю повторить загрузить снимок<br />7.4 Пользователь может повторить попытку загрузки снимка<br />7.5 Use case завершается |
| **Бизнес правила** | BR-05: Доступ только авторизованным пользователям<br />BR-06: Доступ к отдельным снимкам может быть ограничен премиум-подпиской<br />BR-07: Все действия пользователя (включая просмотр снимков) фиксируются системой в истории действий<br />BR-08: Снимку присваивается статус «Просмотрен», если пользователь находился на странице просмотра снимка не менее 30 секунд |

## UC-03: Загрузка, разметка и описание снимка врачом-рентгенологом

| Поле | Описание |
|------|----------|
| **ID** | UC-03 |
| **Название** | Загрузка, разметка и описание снимка врачом-рентгенологом |
| **Акторы** | Врач, Система, Администратор (на этапе модерации снимка) |
| **Предусловия** | Врач авторизован<br />У врача есть интернет-соединение<br />Врач прошел квалификационную проверку<br />У врача есть доступ к разделу добавления снимков |
| **Постусловия (Успех)** | Размеченный снимок с описанием успешно сохранен в системе<br />Снимку присвоен статус "На модерации"<br />Администратор получает уведомление о новом снимке на модерации |
| **Основной поток** | 1. Врач инициирует процесс добавления нового снимка<br />2. Система отображает форму загрузки снимка<br />3. Врач загружает файл рентгенологического снимка<br />4. Система проверяет формат и размер загруженного файла<br />5. Система извлекает технические метаданные из файла (дата исследования, тип исследования и т.д.)<br />6. Если файл корректен, система отображает форму для заполнения метаданных, проведения разметки и описания снимка<br />7. Врач заполняет обязательные данные о снимке: область тела, диагноз, уровень сложности снимка (для студентов)<br />8. Врач выполняет графическую разметку изображения<br />9. Врач добавляет текстовое описание к снимку<br />10. Врач подтверждает сохранение снимка и отправку его на модерацию<br />11. Система проверяет полноту заполнения обязательных данных врачом<br />12. Система сохраняет файл снимка, его разметку и описание<br />13. Система присваивает снимку статус "На модерации"<br />14. Система отправляет уведомление администратору о новом снимке, ожидающем модерации<br />15. Система отображает врачу уведомление об успешной отправке снимка на модерацию |
| **Альтернативный поток** | **A1 - Некорректный формат файла**<br />Точка входа: Во время шага 4<br />4.1 Система определяет, что загруженный файл не соответствует поддерживаемому формату<br />4.2 Система отображает уведомление о неподдерживаемом формате файла<br />4.3 Врач возвращается к шагу 3<br /><br />**A2 - Не заполнены обязательные поля**<br />Точка входа: Во время шага 11<br />11.1 Система обнаруживает незаполненные обязательные поля<br />11.2 Система подсвечивает соответствующие поля и отображает уведомление о необходимости их заполнения<br />11.3 Врач заполняет недостающие поля<br />11.4 Use case возвращается к шагу 11<br /><br />**A3 - Превышен допустимый размер файла**<br />Точка входа: Во время шага 4<br />4.1 Система определяет, что загруженный файл превышает допустимый размер файла<br />4.2 Система отображает уведомление о превышении допустимого размера файла<br />4.3 Врач возвращается к шагу 3 |
| **Исключительный поток** | **E1 - Ошибка загрузки файла**<br />Точка входа: Во время шага 3<br />3.1 Происходит прерывание соединения<br />3.2 Система не может загрузить файл полностью<br />3.3 Система отображает уведомление об ошибке загрузки<br />3.4 Врач может повторить попытку загрузки<br />3.5 Use Case завершается<br /><br />**E2 - Ошибка базы данных при сохранении снимка**<br />Точка входа: Во время шага 12<br />12.1 Возникновение технической ошибки на стороне сервера<br />12.2 Система откатывает операцию сохранения<br />12.3 Система отображает уведомление о технической ошибке<br />12.4 Пользователь может повторить попытку позже<br />12.5 Use Case завершается |
| **Бизнес правила** | BR-09: Снимки загружаются только в виде файлов формата DICOM<br />BR-10: Максимальный размер файла - 100 Мб<br />BR-11: Все снимки проходят обязательную модерацию перед публикацией<br />BR-12: Врач несёт ответственность за корректность диагноза<br />BR-13: Персональные данные пациента должны быть анонимизированы |

## UC-04: Отправка пациентом снимка на консультацию

| Поле | Описание |
|------|----------|
| **ID** | UC-04 |
| **Название** | Отправка пациентом снимка на консультацию |
| **Акторы** | Пациент, Врач, Система |
| **Предусловия** | Пользователь авторизован<br />У пользователя есть интернет-соединение<br />Пациент подтвердил согласие на обработку персональных данных<br />У пациента есть доступ к разделу создания заявки |
| **Постусловия (Успех)** | В системе создается заявка на консультацию<br />Заявке присвоен уникальный номер<br />Снимок и сопроводительная информация сохранены в системе<br />Заявка размещена в очереди на обработку врачами<br />Пациент уведомлен об успешном создании заявки |
| **Основной поток** | 1. Пациент инициирует процесс создания новой заявки на консультацию<br />2. Система отображает форму создания заявки<br />3. Пациент загружает медицинский снимок<br />4. Система проверяет размер и формат загруженного файла<br />5. Пациент заполняет обязательные данные заявки: жалобы, дополнительную информацию, срочность обработки<br />6. Пациент подтверждает отправку заявки<br />7. Система проверяет полноту заполненных данных<br />8. Система создает заявку в базе данных<br />9. Система присваивает статус заявке "Ожидает назначения врача"<br />10. Система отправляет пациенту сообщение об успешном создании заявки<br />11. Система направляет уведомление врачам |
| **Альтернативный поток** | **A1 - Некорректный формат файла**<br />Точка входа: Во время шага 4<br />4.1 Система определяет, что загруженный файл не соответствует формату<br />4.2 Система уведомляет пациента о неподдерживаемом формате файла<br />4.3 Пациент возвращается к шагу 3<br /><br />**A2 - Превышен допустимый размер файла**<br />Точка входа: Во время шага 4<br />4.1 Система определяет, что загруженный файл превышает допустимый размер файла<br />4.2 Система отображает уведомление о превышении допустимого размера файла<br />4.3 Пациент возвращается к шагу 3<br /><br />**A3 - Нет свободных врачей**<br />Точка входа: Во время шага 11<br />11.1 Система не находит доступных врачей<br />11.2 Система присваивает заявке статус "В очереди"<br />11.3 Система уведомляет пациента о задержке подбора подходящего врача<br /><br />**A4 - Не заполнены обязательные поля**<br />Точка входа: Во время шага 7<br />7.1 Система обнаруживает незаполненные обязательные поля<br />7.2 Система подсвечивает соответствующие поля и отображает уведомление о необходимости их заполнения<br />7.3 Пациент заполняет недостающие поля<br />7.4 Use case возвращается к шагу 7 |
| **Исключительный поток** | **Сбой при создании заявки**<br />Точка входа: Во время шага 8<br />8.1 В системе возникает ошибка базы данных<br />8.2 Система отображает пациенту уведомление о технической ошибке<br />8.3 Пациент может повторить попытку позже<br />8.4 Use Case завершается |
| **Бизнес правила** | BR-14: Максимальный размер файла - 100 Мб<br />BR-15: Поддерживаемые форматы снимков - DICOM, PNG, JPG<br />BR-16: Заявка хранится в архиве не менее 5 лет<br />BR-17: Медицинские требования хранятся с требованиями ФЗ «О персональных данных»<br />BR-18: Врач может взять заявку на консультацию из очереди доступных заявок |
