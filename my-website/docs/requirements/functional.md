## UC-01: Регистрация пользователя

Пользователь создает учётную запись, выбирая роль. Для врачей требуется загрузка подтверждающего документа.

<details>
<summary>📋 Детали сценария</summary>

| Поле | Описание |
|------|----------|
| **ID** | UC-01 |
| **Акторы** | Незарегистрированный пользователь, Система |
| **Предусловия** | Пользователь не зарегистрирован, есть интернет, находится на странице регистрации |
| **Постусловия** | Учетная запись создана |
| **Основной поток** | 1. Заполнение формы (логин, пароль, email, роль)<br/>2. Подтверждение согласия с политикой конфиденциальности<br/>3. Инициирование регистрации<br/>4. Проверка данных (формат email, уникальность, надежность пароля)<br/>5. Сохранение в БД<br/>6. Уведомление об успешной регистрации |
| **Альтернативный поток** | **Регистрация врача**: на шаге 1 добавляется поле загрузки документа о квалификации. **Просмотр политики**: на шаге 2 переход по ссылке, отображение окна, закрытие. |
| **Исключительный поток** | **Ошибка сети**: на шаге 3 сбой передачи, уведомление, повтор. **Email уже зарегистрирован**: на шаге 4 обнаружен дубликат, предложение восстановить пароль. |
| **Бизнес-правила** | BR-01: Пароль ≥ 8 символов. BR-02: Email уникален. BR-03: Роль фиксируется при регистрации. BR-04: Для врача — документ PDF/PNG до 5 МБ. |

</details>


## UC-02: Просмотр рентгеновского снимка из учебной базы

Студент открывает учебный снимок с разметкой и описанием для развития насмотренности.

<details>
<summary>📋 Детали сценария</summary>

| Поле | Описание |
|------|----------|
| **ID** | UC-02 |
| **Акторы** | Студент, Система |
| **Предусловия** | Авторизован, есть интернет, в базе имеются опубликованные снимки, находится на главной странице |
| **Постусловия** | Снимок отображён с разметкой и описанием. Факт просмотра зафиксирован. При просмотре ≥30 сек — статус «Просмотрен». |
| **Основной поток** | 1. Открытие раздела «База снимков»<br/>2. Отображение категорий<br/>3. Выбор категории<br/>4. Отображение списка снимков<br/>5. Выбор конкретного снимка<br/>6. Проверка прав доступа<br/>7. Отображение снимка с разметкой и описанием<br/>8. Просмотр<br/>9. Фиксация в истории<br/>10. Присвоение статуса «Просмотрен» через 30 сек |
| **Альтернативный поток** | **Нет доступа**: на шаге 6 — снимок требует премиум-подписки, уведомление, предложение оформить подписку или вернуться к списку. |
| **Исключительный поток** | **Ошибка отображения**: на шаге 7 — сбой загрузки, уведомление, возможность повтора. |
| **Бизнес-правила** | BR-05: Только авторизованные пользователи. BR-06: Доступ к отдельным снимкам по премиум-подписке. BR-07: Все действия фиксируются в истории. BR-08: Статус «Просмотрен» после 30 сек. |

</details>


## UC-03: Загрузка, разметка и описание снимка врачом

Врач загружает медицинский снимок, выполняет разметку, добавляет описание и отправляет на модерацию.

<details>
<summary>📋 Детали сценария</summary>

| Поле | Описание |
|------|----------|
| **ID** | UC-03 |
| **Акторы** | Врач, Система, Администратор (модерация) |
| **Предусловия** | Врач авторизован, прошёл квалификационную проверку, есть доступ к разделу добавления снимков |
| **Постусловия** | Размеченный снимок с описанием сохранён, статус «На модерации», администратор уведомлён |
| **Основной поток** | 1. Инициирование добавления снимка<br/>2. Отображение формы загрузки<br/>3. Загрузка файла<br/>4. Проверка формата и размера<br/>5. Извлечение технических метаданных<br/>6. Отображение формы разметки и описания<br/>7. Заполнение обязательных полей (область, диагноз, уровень сложности)<br/>8. Графическая разметка<br/>9. Текстовое описание<br/>10. Подтверждение сохранения<br/>11. Проверка полноты данных<br/>12. Сохранение файла, разметки и описания<br/>13. Присвоение статуса «На модерации»<br/>14. Уведомление администратора<br/>15. Уведомление врача об успехе |
| **Альтернативный поток** | **Некорректный формат**: на шаге 4 — ошибка, возврат к шагу 3. **Незаполненные поля**: на шаге 11 — подсветка полей, возврат. **Превышен размер**: на шаге 4 — уведомление, возврат к шагу 3. |
| **Исключительный поток** | **Ошибка загрузки файла**: на шаге 3 — прерывание соединения, уведомление, завершение. **Ошибка БД**: на шаге 12 — откат, уведомление, завершение. |
| **Бизнес-правила** | BR-09: Только DICOM. BR-10: Максимальный размер 100 МБ. BR-11: Обязательная модерация. BR-12: Врач несёт ответственность за диагноз. BR-13: Персональные данные анонимизируются. |

</details>


## UC-04: Отправка пациентом снимка на консультацию

Пациент создает заявку на диагностику, загружает медицинский снимок и ожидает заключения врача.

<details>
<summary>📋 Детали сценария</summary>

| Поле | Описание |
|------|----------|
| **ID** | UC-04 |
| **Акторы** | Пациент, Врач, Система |
| **Предусловия** | Пациент авторизован, дал согласие на обработку ПД, есть доступ к созданию заявки |
| **Постусловия** | Заявка создана с уникальным номером, снимок и данные сохранены, заявка в очереди, пациент уведомлён |
| **Основной поток** | 1. Инициирование создания заявки<br/>2. Отображение формы<br/>3. Загрузка снимка<br/>4. Проверка размера и формата<br/>5. Заполнение обязательных полей (жалобы, дополнительная информация, срочность)<br/>6. Подтверждение отправки<br/>7. Проверка полноты данных<br/>8. Создание заявки в БД<br/>9. Присвоение статуса «Ожидает назначения врача»<br/>10. Уведомление пациента<br/>11. Уведомление врачей |
| **Альтернативный поток** | **Некорректный формат**: на шаге 4 — уведомление, возврат к шагу 3. **Превышен размер**: аналогично. **Нет свободных врачей**: на шаге 11 — статус «В очереди», уведомление пациента. **Незаполненные поля**: на шаге 7 — подсветка, возврат. |
| **Исключительный поток** | **Сбой при создании**: на шаге 8 — ошибка БД, уведомление, завершение. |
| **Бизнес-правила** | BR-14: Максимальный размер 100 МБ. BR-15: Форматы DICOM, PNG, JPG. BR-16: Хранение заявки ≥ 5 лет. BR-17: Соответствие ФЗ «О персональных данных». BR-18: Врач берёт заявку из очереди. |

</details>
